This section contains requirements and other information concerning the use of PUF Invoice Response in France.

==== Payment reporting

In France it is a requirement to report received payments for certain transactions to the Tax Authority. The PUF Invoice Response format has been extended with additional elements relating to payment amounts to facilitate this. The full list of elements is available in <<_pufresponseextension>>.


===== Amount Type

The clarification type code (``puf:Clarification/puf:TypeCode``) is used to indicate what the specified amounts mean. In France, the following list is applicable:

.puf:TypeCode (amount type) values applicable in France
|===
|Code |Definition

|`MEN`
|Amount collected (incl. tax)

|`MPA`
|Amount paid

|`RAP`
|Remaining to pay (in case of partial payment)

|`ESC`
|Discount granted

|`RAB`
|Rebate granted

|`REM`
|Reduction granted

|`MAP`
|Net Amount Approved

|`MAPTTC`
|Gross Amount Approved (incl. tax)

|`MNA`
|Net Amount NOT Approved

|`MNATTC`
|Gross Amount NOT Approved (incl. tax)
|===

NOTE: *MEN* (with accompanying ``puf:ValueAmount``/``puf:ValueAmountCurrency`` and ``puf:ValuePercent``) is the only type *required* by the Tax Authority, but the other type codes (``puf:TypeCode``) and their values are useful to convey additional information to the receiving party and can be used *in addition to* MEN.

NOTE: Each ``puf:Clarification`` carries a single ``puf:ValueAmount``; whether it represents a net (excl. tax) or gross (incl. tax) amount is conveyed by the ``puf:TypeCode`` (e.g. *MAP*/*MNA* are net, *MEN*/*MAPTTC*/*MNATTC* are gross).

==== Requested Action

It is possible to request the recipient of the PUF Invoice Response to perform a certain action (generally to resolve a dispute). A common example scenario could be one where the Buyer requests that the Seller issues a new invoice, as the current one has incorrect data. 

See the section <<_cacresponse>> for additional details. 

In France, the action codes will follow the Peppol list https://docs.peppol.eu/poacc/upgrade-3/codelist/OPStatusAction/[OPStatusAction^]. 

NOTE: *Only* the Action Code list follows Peppol. The Reason code list is unique to France and can be found in the <<_status_reason, Status Reason section above>>.

==== Compliance and clearance handling extensions

The document-level extensions `puf:ComplianceExtension` (`puf:ComplianceActivatedIndicator`, `puf:SkipClearanceIndicator`) and `puf:ExternalDeliveryType`, described in the Extensions chapter, are generic but are currently used in France as described below.

===== Skipping clearance for optional statuses (`puf:SkipClearanceIndicator`)

The most important use case in France is the *Payment Received* status (_EncaissĂ©e_, French CDAR code 212, response code `PAYMENT_RECEIVED`). As a rule, these statuses are also transmitted to the Clearance Authority (PPF) for approval. In certain situations, however, the supplier has *no regulatory obligation* to report this status â€” yet may still wish to send it for *business* reasons, for example because the buyer asks for it.

To support this, set `puf:SkipClearanceIndicator` to `true`. Pagero Online then processes the document as usual but *skips the clearance / PPF transmission step*.

*Example* +
_Payment Received (EncaissĂ©e) status that should be processed but not sent to clearance._
[source,xml]
----
<ApplicationResponse>
    <ext:UBLExtensions>
        <ext:UBLExtension>
            <ext:ExtensionURI>urn:pagero:ExtensionComponent:1.0:PageroExtension:ComplianceExtension</ext:ExtensionURI>
            <ext:ExtensionContent>
                <puf:PageroExtension>
                    <puf:ComplianceExtension>
                        <puf:SkipClearanceIndicator>true</puf:SkipClearanceIndicator>
                    </puf:ComplianceExtension>
                </puf:PageroExtension>
            </ext:ExtensionContent>
        </ext:UBLExtension>
    </ext:UBLExtensions>
    <!-- Code omitted for clarity; e.g. a PAYMENT_RECEIVED (212) document status -->
</ApplicationResponse>
----

===== `puf:ComplianceActivatedIndicator` and `puf:ExternalDeliveryType`

`puf:ComplianceActivatedIndicator` controls whether the document is processed within the French e-invoicing reform. When `true`, Pagero â€” acting as the accredited platform (PA) â€” applies the stricter compliance checks and validations the reform requires; when `false` or absent, the document is processed outside that regime and those checks are not applied. Primarily used for e.g. B2C or cross-border documents.

`puf:ExternalDeliveryType` requests that Pagero Online process the document without sending it onward: `PROCESS_WITHOUT_DELIVERY` processes the document but does not deliver it, while `PROCESS_WITHOUT_ROUTING` processes it but neither routes nor delivers it. Primarily used when you want Pagero to archive a file but not distribute it.
